C++ Introduction

C++ is a powerful, general-purpose programming language developed by Bjarne Stroustrup as an extension of the C language. It is known for its high performance and control over system resources and memory, making it suitable for developing a wide range of applications, including operating systems, game engines, and high-performance computing software.

History

C++ was created by Bjarne Stroustrup at Bell Laboratories in 1979. It was initially named "C with Classes" as it extended the C programming language with object-oriented features,influenced by the Simula 67 language. Stroustrup aimed to combine the efficiency of C with the organizational benefits of object-oriented programming for large-scale projects, particularly in simulating distributed systems.

Key milestones in C++ history

  • 1979 Bjarne Stroustrup begins development of "C with Classes" at Bell Labs.
  • 1983 "C with Classes" is renamed C++, symbolizing its evolution from C using the increment operator.
  • 1985 The first commercial version of C++ is released, and "The C++ Programming Language" by Stroustrup is published.
  • 1989 C++ 2.0 is released, introducing features like protected and static members, and multiple inheritance.
  • 1998 The first international standard for C++, C++98, is released, including the Standard Template Library (STL).
  • 2003 C++03 is released, primarily a maintenance update to C++98.
  • 2011 C++11, a major revision, is released, introducing numerous new features like lambda expressions, nullptr, and auto.
  • 2014 C++14 is released, providing minor updates and bug fixes to C++11.
  • 2017 C++17 is released, a significant update that added new features and removed obsolete ones.
  • 2020 C++20 is released, introducing concepts, modules, and coroutines, among other features.
  • 2024 C++23, the latest version, is launched, bringing features like pattern matching and expanded support for ranges.

C++ Working

C++ is a compiled, object-oriented programming language that enables the creation of high-performance applications. Its working mechanism involves several key steps

Writing Source Code

A developer writes C++ code in a text editor or an Integrated Development Environment (IDE) like Visual Studio, CodeBlocks, or VS Code. This code is human-readable instructions for the computer.

Compilation

The C++ source code cannot be directly executed by the computer. It must first be translated into machine-understandable instructions (machine code). This process is called compilation and is performed by a C++ compiler (e.g., GCC, Clang, MSVC). Preprocessing: The compiler first handles preprocessor directives (lines starting with #), such as #include for including header files, which effectively paste the content of specified files into the current source file. Compilation to Object Files: Each .cpp source file is then compiled into an object file (e.g., .obj on Windows). This object file contains machine code for that specific source file but is not yet a complete executable program.

Linking

The linker takes all the object files generated from the different source files, along with any necessary library files (pre-compiled code that provides common functionalities), and combines them into a single, executable binary file (e.g., .exe on Windows).

Execution

Once the executable file is created, it can be run by the operating system. The operating system loads the program into memory and begins executing the machine code instructions, leading to the program's intended behavior, such as displaying output in a console window or interacting with a Graphical User Interface (GUI).

C++ Variables

In C++, variables are named storage locations in computer memory used to hold data values. They are fundamental building blocks of any C++ program, allowing data to be stored, accessed, and manipulated during program execution.

Declaration

Before using a variable, it must be declared. This involves specifying the variable's data type and giving it a name (identifier). The data type determines the kind of data the variable can store (e.g., integers, floating-point numbers, characters, booleans). 

    int age;         // Declares an integer variable named 'age'
    double price;    // Declares a double-precision floating-point variable named 'price'
    char initial;    // Declares a character variable named 'initial'
    bool isActive;   // Declares a boolean variable named 'isActive'

Initialization

Variables can be assigned an initial value during declaration or later in the program. 

    int count = 10;   // Declares and initializes 'count' to 10
    double temperature;
    temperature = 25.5; // Assigns a value to 'temperature' later

Naming Conventions:

Variable names should be descriptive and follow C++ naming conventions (e.g., camelCase for most variables, all caps with underscores for constants). They cannot start with a number and cannot contain special characters other than underscores.

Scope

Variables have a defined scope, which determines where they can be accessed within a program (e.g., local variables within a function, global variables accessible throughout the program).

C++ Data Types

C++ data types categorize the kind of values a variable can hold and determine the operations that can be performed on them. They are broadly classified into three categories

Built-in (Primitive) Data Types

These are fundamental types pre-defined by the C++ language.

  • int: Stores whole numbers (integers), e.g., 5, -100.
  • float: Stores single-precision floating-point numbers (numbers with decimals), e.g., 3.14f.
  • double: Stores double-precision floating-point numbers, offering higher precision than float, e.g., 3.14159.
  • char: Stores a single character, e.g., 'a', 'Z', '5'.
  • bool Stores boolean values, either true or false.
int num=20;
float pi=3.14;

char name='A'

Derived Data Types

These are built upon or derived from the primitive data types.

  • Arrays: Collections of elements of the same data type, accessed by an index, e.g., int numbers[5];.
  • Pointers: Variables that store memory addresses of other variables, e.g., int *ptr;.
  • References: Aliases for existing variables, providing another name to access the same memory location, e.g., int &ref = x;.
  • Functions: Blocks of code that perform specific tasks and can return values of a particular data type.

User-Defined Data Types:

These are created by the programmer to represent more complex data structures.

  • Classes : Blueprints for creating objects, encapsulating data (member variables) and functions (member methods).
  • Structures (struct): Similar to classes but with public members by default, used for grouping related data.
  • Unions (union): Allow different data types to share the same memory location, but only one member can be active at a time.
  • Enumerations (enum): Define a set of named integer constants.

C++ Functions

In C++, a function is a named block of code designed to perform a specific task. Functions are fundamental for organizing code, promoting reusability, and improving readability.

Declaration and Definition

Declaration: Specifies the function's return type, name, and parameters (function signature). It informs the compiler about the function's existence before its full definition.

Definition: Provides the actual implementation of the function's logic within curly braces.

    return_type function_name(parameter_type1 parameter_name1, parameter_type2 parameter_name2, ...) {
        // Function body: statements to be executed
        // return value; (if return_type is not void)
    }

Return Type

Specifies the data type of the value the function will return. void indicates that the function does not return any value.

Parameters

Placeholders for values passed into the function when it is called (arguments).They can be passed by value (a copy is made) or by reference (direct access to the original variable). Default parameter values can be assigned.

Function Name

An identifier used to call or invoke the function.

Function Body

The enclosed block of statements that define the function's operations.

Calling a Function

Functions are executed only when called by their name followed by parentheses and arguments (if any). Example: myFunction(argument1, argument2);

Types of Functions

  • User-defined functions: Created by the programmer to perform specific tasks.
  • Library functions: Pre-defined functions available in C++ standard libraries (e.g., functions in like sqrt(), pow()).
  • main() function: The entry point of every C++ program, where execution begins.

C++ Loops

Loops in C++ are control flow statements that enable the repeated execution of a block of code based on a specified condition. They are fundamental for automating repetitive tasks and iterating over collections of data. C++ offers three primary types of loops:

for loop

Used when the number of iterations is known or can be determined before the loop begins. 

        for (int i = 0; i < 5; i++) {
            // This code will execute 5 times
        }

while loop

Used when the number of iterations is unknown and the loop continues as long as a specified condition remains true. 

        int count = 0;
        while (count < 5) {
            // This code will execute as long as count is less than 5
            count++;
        }

do.. whole loop

Similar to the while loop, but guarantees at least one execution of the loop body before checking the condition.

        int choice;
        do {
            // This code will execute at least once
            // Prompt for user input
        } while (choice != 0); // Loop continues until user enters 0

Control Statements within Loops:

  • break:Terminates the loop immediately, transferring control to the statement following the loop.
  • continue:Skips the current iteration of the loop and proceeds to the next iteration.

C++ Objects

In C++, an object is an instance of a class. While a class serves as a blueprint or a template defining the structure and behavior (data members and member functions) of a particular type, an object is a concrete realization of that class.

Key characteristics of C++ objects

Instance of a Class:

An object is created from a class, meaning it adheres to the structure and rules defined by that class.

Memory Allocation:

When an object is created, memory is allocated to store its data members. This is in contrast to a class, which is a logical construct and does not occupy memory until an object is instantiated.

State and Behavior:

Objects possess both a "state" (represented by the values of their data members) and "behavior" (represented by the actions performed by their member functions).

Runtime Entity:

Objects are created and exist during the program's execution, making them runtime entities.

Access to Class Members:

Objects provide the means to access and manipulate the data members and invoke the member functions defined within their class.

Example

 #include 
  #include 

    class Car {
    public:
    std::string brand;
    std::string model;
    int year;

    void displayInfo() {
        std::cout << "Brand: " << brand << ", Model: " << model << ", Year: " << year << std::endl;
    }
};

    int main() {
    // Creating an object of the Car class
    Car myCar; 

    // Accessing and setting object's data members
    myCar.brand = "Toyota";
    myCar.model = "Camry";
    myCar.year = 2022;

    // Calling object's member function
    myCar.displayInfo(); 

    return 0;
}

C++ Classes

In C++, a class is a user-defined data type that serves as a blueprint for creating objects. It encapsulates data members (variables) and member functions (methods) within a single unit, promoting organization and reusability in code.

Key Concepts of C++ Classes

Encapsulation:

Classes bundle data and the functions that operate on that data together, hiding internal implementation details from external access.

Data Members:

These are variables declared within a class that store the state or attributes of an object created from that class.

Member Functions:

These are functions declared within a class that define the behaviors or operations that can be performed on the objects of that class.

Access Specifiers:

C++ classes use private, public, and protected access specifiers to control the visibility and accessibility of their members.

Objects:

An object is an instance of a class, created based on the class's blueprint. Each object has its own set of data members and can invoke the member functions defined in its class.

Constructors and Destructors:

Constructors: Special member functions that are automatically called when an object of the class is created. They are used to initialize the object's data members.

Destructors: Special member functions that are automatically called when an object is destroyed. They are used to perform cleanup operations, such as releasing dynamically allocated memory.

Example of a C++ Class:

  class Car {
public: // Public members are accessible from outside the class
    std::string brand;
    std::string model;
    int year;

    // Member function to display car information
    void displayInfo() {
        std::cout << "Brand: " << brand << ", Model: " << model << ", Year: " << year << std::endl;
    }

    // Constructor
    Car(std::string b, std::string m, int y) : brand(b), model(m), year(y) {}
};

// Creating an object of the Car class
Car myCar("Toyota", "Camry", 2023);
myCar.displayInfo(); // Calling a member function

JavaScript Async

In JavaScript, "async" is a keyword used to define an asynchronous function. Asynchronous functions are designed to handle operations that might take time to complete, such as fetching data from a server or reading a file, without blocking the execution of the rest of the program. This ensures that the user interface remains responsive and the application doesn't freeze during these long-running tasks.

Async keyword

Placed before the function keyword (or arrow function syntax) to declare an asynchronous function. An async function automatically returns a Promise. The value returned from an async function resolves the Promise, while any error thrown within it rejects the Promise.

 async function fetchData() {
  // Asynchronous operations here
      return "Data fetched successfully!";
    }

Await keyword

Can only be used inside an async function. Used to pause the execution of the async function until a Promise settles (either resolves or rejects). When await is used with a Promise, the function execution pauses until that Promise is fulfilled, and then resumes with the resolved value.

    async function getUser() {
      const response = await fetch('https://api.example.com/users/1');
      const userData = await response.json();
      console.log(userData);
    }

Error Handling:

try...catch blocks are used within async functions to handle errors (rejected Promises).

  async function getSafeData() {
      try {
        const response = await fetch('https://api.example.com/invalid-url');
        const data = await response.json();
        console.log(data);
      } catch (error) {
        console.error("Error fetching data:", error);
      }
    }

JavaScript DOM

The Document Object Model (DOM) is a programming interface for HTML, XML, and SVG documents. It represents the structure of a document as a logical tree, where each node in the tree is an object representing a part of the document, such as an element, attribute, or text content. While the DOM is not part of the core JavaScript language, it is a crucial Web API that allows JavaScript to interact with and manipulate the content, structure, and style of web pages.

Tree Structure

The browser creates a DOM tree when a web page loads, representing the hierarchical relationship between elements.

Nodes and Objects

Each part of the document (elements, attributes, text) is represented as a node object in the DOM tree, with properties and methods for interaction.

Accessing Elements

JavaScript provides methods to select and access specific elements in the DOM, such as document.getElementById() ,document.querySelector(), and document.getElementsByClassName().

Manipulation

Once an element is accessed, JavaScript can modify its properties (e.g., innerHTML, style), add or remove elements, and change attributes.

Example

// Select an element by its ID
const demoParagraph = document.getElementById("demo");

// Change the text content of the paragraph
demoParagraph.innerHTML = "Hello World from JavaScript!";

// Add a CSS class to the paragraph
demoParagraph.classList.add("highlight");
