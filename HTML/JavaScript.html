JavaScript Introduction

JavaScript, often abbreviated as JS, is a programming language and core technology of the world wide web, alongside HTML and CSS.99% of websites use JavaScript on the client side for webpage behavior. Web browsers have a dedicated JavaScript Engine that execute the client code.

History

JavaScript was invented by Brendan Eich in 1995. It was developed for Netscape 2 and became the ECMA-262 standard in 1997. After Netscape handed JavaScript over to ECMA, the Mozilla foundation continued to developed JavaScript for Firefox browsers.

JavaScript Working

JavaScript code is executed by a JavaScript engine, which is program designed to interpret and execute JavaScript code. These engines are found in web browsers and also in server side runtimes like Node.js.

Parsing and AST Generation

The JavaScript engine first parses the code breaking it down into a structured representation called an Abstract Syntax Tree (AST). This tree represents the grammatical strucure of the code.

Interpretation and Bytecode Generation

The AST is then interpreted and in many modren engines, bytecode is generated. Bytecode is a low-level representation of the code that is more efficient for the machine to process than the raw source code.

Just-In-Time Compilation

To optimize performance, JavaScript engines employ Just-In-Time Compilation. This process combines interpretation with Compilation. Frequently executed or hot code paths are identified and compiled into highly optimized machine code, which can be executed directly by thr computer hardware.

Memory Management

JavaScript engines also manage memory. Objects and data structures with longer lifetimes are stored in Heap, which is dynamically managed. A Garbage collecter automatically identifies and reclaims memory that is no longer being used by the program.

Variables

In JavaScript, variables are named containers used to store data values. They are essential for holding information that can be accessed and manipulated within a program. JavaScript offers three primary keywords for declaring variables var, let, and const.

Var

This is the older way of declaring variables in JavaScript. var declarations are function-scoped or globally-scoped. This means a variable declared with var inside a function is accessible anywhere within that function, and if declared outside any function, it's accessible globally. Variables declared with var can be re-declared and re-assigned within the same scope without error.

Let

This is the older way of declaring variables in JavaScript. Introduced in ES6 (ECMAScript 2015), let is the modern and recommended way to declare variables. let declarations are block-scoped. This means a variable declared with let is only accessible within the block (e.g., inside an if statement, for loop, or any curly braces {}) where it was defined. Variables declared with let can be re-assigned but cannot be re-declared within the same scope

Const

Also introduced in ES6, const is used to declare variables whose values are intended to remain constant (immutable) after initial assignment. Like let, const declarations are block-scoped. Variables declared with const must be initialized at the time of declaration and cannot be re-assigned after their initial value is set. However, for const variables holding objects or arrays, the contents of the object or array can still be modified, but the variable itself cannot be pointed to a different object or array.

var globalVar = "I'm global";
function exampleVar() {
  var functionVar = "I'm function-scoped";
  console.log(globalVar); // Accessible
  console.log(functionVar); // Accessible
}
exampleVar();
// console.log(functionVar); // Error: functionVar is not defined outside the function

// Using let (block-scoped)
let blockScope = "Outside block";
if (true) {
  let blockScope = "Inside block"; // This is a new variable, not redeclaration
  console.log(blockScope); // Outputs: Inside block
}
console.log(blockScope); // Outputs: Outside block

// Using const (block-scoped and immutable value)
const PI = 3.14159;
// PI = 3.14; // This would cause an error: Assignment to constant variable.

const myArray = [1, 2, 3];
myArray.push(4); // Allowed: Modifying array content
// myArray = [5, 6]; // Error: Assignment to constant variable.

JavaScript Data Types

JavaScript categorized data into two types: Primitive and Reference

Primitive Data Types

These are immutable values, meaning their value cannot be changed after creation. There are seven priitive data types in JavaScript

String: Represents textual data, enclosed in single quotes, double quotes or backticks.

let name='Alice';
let greeting=`Hello, ${name}`

Number: Represents both integer and floating-point numbers. Includes special values like Infinity and NaN. 

let age=30;
let price=9.99

Boolean:Represents logical entities, with values true or false

let isactive=true;
let isloggedIn=false

Undefined: Represents a variable that has been declared but not yet assigned a value.

let unassignedVariable; 
// unassignedVariable is Undefined

Null: Represents the intentational absense of any object value. It is a primitive value but typeof null rerurns object due to a historical quirk

let emptyValue=null;
let empty=null

Bigint: Represents whole numbers larger than 2 53 -1, which is the maximum safe integer for the Number type. Appended with an n.

let largeNumber=9007199254740991n

Symbol: A unique and immutable primitive value, often used as obejct property key to avoid naming collision. 

const id= Symbol('uniqueId')

JavaScript Functions

JavaScript functions are fundamental blocks of code designed to perform specific tasks. They allow for organization, reusability, and modularization of code. Functions can take inputs (parameters), perform actions, and return outputs.

Function Expression.

A function defined as part of an expression, often assigned to a variable. It can be named or anonymous. 

const functionName = function(parameter1, parameter2)
 {
// code to be executed
return result;    };

Arrow Functions (ES6+).

A concise syntax for writing function expressions, particularly useful for shorter functions and when dealing with this context. 

const functionName = (parameter1, parameter2) => {
// code to be executed
return result;
};

// For single-expression functions, the return keyword and curly braces can be omitted
const add = (a, b) => a + b;

Invoking Functions:

Functions are executed when they are "invoked" or "called." This can happen: When an event occurs (e.g., a user clicks a button). When explicitly called from other JavaScript code

JavaScript Loops

Loops in JavaScript are control flow statements used to repeatedly execute a block of code as long as a specified condition remains true. They are essential for automating repetative tasks and iterating over collections of data.

for loop

Used for iterating a specific number of times or iterating over elements of an array. 

for (let i=0; i<=5 ;i++){
    console.log(i)  //prints 0,1,2,3,4,5
}

while loop

Executes a block of code as long as a condition remains true. The condition is checked before each iteration. 

let i=0;
while(i<=5){
    console.log(i) //prints 0,1,2,3,4,5
    i++
}

do.. whole loop

Similar to while loop but the code block is executed before the condition is checked. The loop then continues as long as the condition is ture.

let i=0;
  do{
       console.log(i) //prints 0,1,2,3,4,5
        i++
    } while(i<=5)

for.. of loop

Used to iterate over iteratable objects like (array, String or Maps) and access the values of each element

const colors=['red','green','yellow']
  for(const color of colors){
    console.log(color)  //prints  red, green , yellow
  }
